# v2.1 Scope — Profile-Guided SIMD + Live Demo Deployment

Status: **Implemented.** Phase 8 (live demo deploy) merged; Phase 7 (profile-guided
SIMD) implemented on `claude/phase7-simd` — gate measured ([11b](research/11b-profile-results.md)),
`wide::f32x8` geometry kernel byte-identical to scalar (FR-7.4) at ~1.5–1.6× on the
geometry stage (NFR-13). De-risking research: [11](research/11-simd-profiling.md),
[12](research/12-github-pages-deploy.md). Target version: **2.1.0**. Builds on v2.0
(browser frontend + parallel rasterizer).

## 1. Theme

Two complementary goals: make the engine **faster** (deliver the SIMD piece deferred from
v2.0, now profile-guided) and make it **visible** (a live, auto-deployed, in-CI-tested demo).

1. **Profile-guided SIMD** (Phase 7) — vectorize the *actually hot* stage on stable Rust,
   keeping output **byte-identical** to the scalar path.
2. **Live demo deployment** (Phase 8) — auto-publish the WASM demo to GitHub Pages on merge,
   with a CI smoke test so the demo is a genuine, testable artifact.

## 2. Key decisions

| # | Decision | Rationale (report) |
|---|---|---|
| W1 | **SIMD effort is profile-gated**: a samply/perf profile of the 100k-tri @ 400×200 frame must confirm the geometry stage (Mat4×Vec4 transforms + edge-function float setup) dominates *before* writing the kernel. If it doesn't, stop and re-scope. | 11 — prior insight is a hypothesis until measured |
| W2 | **SIMD crate: `wide`** (stable; fixed `f32x8`/`f32x4`; SSE2→AVX2→NEON→wasm-simd128 with scalar fallback). `std::simd` is still nightly (excluded); `glam` is horizontal-only (not adopted); `pulp`/`multiversion` deferred. | 11 |
| W3 | **Vectorize the geometry stage first**: vertex positions → SoA/AoSoA, batch the transform 8 vertices at a time with `wide::f32x8`. The integer `orient2d` rasterizer math stays exact and untouched. | 11 |

| W5 | **`simd` feature flag** (off by default initially; opt-in), mirroring the `parallel` feature. Scalar fallback always builds. The wasm build enables it via `-C target-feature=+simd128`. | 11 |
| W6 | **Cost tracking**: add `iai-callgrind` per-stage instruction-count benches (deterministic, CI-gateable) alongside criterion wall-clock for the speedup NFR. | 11 |

## 3. Phase 7 — Profile-guided SIMD

**Deliverable:** a `wide::f32x8` AoSoA transform kernel behind a `simd` feature that makes
the geometry stage measurably faster while producing byte-identical frames.

| ID | Requirement |
|---|---|
| FR-7.1 | **Profile gate**: a documented profile of `solid_100k_tri @ 400×200` confirming the geometry stage dominates. Committed as `docs/research/11b-profile-results.md`. **DONE — see 11b.** Result: geometry stage dominates (61% wall), **but** the per-vertex `Mat4 × Vec4` transform is only ~3.7% of instructions / ~10% of frame; the cost is the **per-triangle** projection/shading + libm round/floor/ceil. FR-7.2 re-scoped accordingly (below). |
| FR-7.2 | **(Re-scoped per FR-7.1.)** AoSoA the *whole per-triangle geometry stage* and batch **8 triangles per `wide::f32x8` lane**: perspective divides, `face_normal` (cross + sqrt-normalize), shading dot/clamp, the `screen_vertex` viewport map, `floor`/`ceil` row span, and the vertex `Mat4 × Vec4` transform — all plain `*`/`+`/`/`/`sqrt`, **no `mul_add`**, scalar add order replayed. The integer `orient2d` rasterizer (`fill_triangle`) stays untouched (W3/W4). |
| FR-7.3 | Wire the kernel into the geometry stage (`prepare_mesh`) behind a `simd` feature; scalar path unchanged and selected when the feature is off. |
| FR-7.4 | **Byte-identical parity test**: a SIMD-vs-scalar framebuffer equality test (the FR-6.3 pattern) on a large mesh; plus the existing golden frames pass with `--features simd`. |
| FR-7.5 | **Partial.** The criterion `solid_100k_tri` bench reads the scalar-vs-simd wall-clock (via the `simd` feature flag), and the `prof_stage_split` harness reports the per-stage split. The `iai-callgrind` deterministic per-stage instruction-count gate is **deferred** (follow-up): correctness is fully guarded by the cross-target byte-identical parity test, and `examples/profile_frame` gives instruction counts under callgrind on demand. |

## 5. Implementation plan (ordered)

1. **Phase 8 first (low-risk, high-visibility):** add `deploy.yml`, `web/.nojekyll`, `scripts/smoke.mjs`, the `demo-smoke` CI job; flip the repo Pages source. Ship the live demo early so v2.1 has a visible artifact independent of the SIMD outcome.
2. **FR-7.1 profile gate:** measure; commit the profile results. Decision point.
3. **FR-7.2–7.4:** SoA layout + `f32x8` kernel + feature wiring + parity test (golden frames re-run, must stay identical — *no* re-approval expected, unlike FR-6.0).
4. **FR-7.5 + NFR matrix:** iai-callgrind benches, CI `simd` feature/target jobs, measure NFR-13.

Splitting this way means the demo (Phase 8) is shippable even if the profile (FR-7.1) sends SIMD back for re-scoping.

## 7. Risks

1. **Profile disproves geometry dominance** (FR-7.1) → the kernel targets the wrong stage. *Mitigation:* W1 gates the whole effort on the measurement; Phase 8 ships regardless.
2. **Byte-identical parity breaks** on some target. *Mitigation:* W4's three conditions (no FMA, scalar add order, SSE2+) are well-sourced; the parity test runs per-target in CI; an accidental `mul_add` is the only realistic regression and is grep-able.
3. **SIMD speedup below 1.5×** (bandwidth-bound). *Mitigation:* NFR-13 is set at the honest research figure; if `perf stat` shows bandwidth-bound, document and accept the measured number rather than chase it.
4. **First Pages deploy fails** (Source not set). *Mitigation:* D2 checklist makes the one-time manual step explicit.
5. **Stale wasm cache** on redeploy. *Mitigation:* FR-8.2 cache-bust query.
