# v2.2 Scope — Mobile Compatibility & Performance

Status: **Scoped — research complete** ([14](research/14-mobile-compatibility.md) mobile
compatibility; references [10](research/10-wasm-threads-hosting.md) wasm threads). Target
version: **2.2.0**. Builds on v2.1 (SIMD geometry + live demo).

## 1. Theme

Make the live demo **load everywhere again** and run **smoothly on mobile iOS** — *without
changing what the engine renders or how it deploys*. Every change is presentation-layer or
build/loader; the software rasterizer, its byte-identical output, the data-only WASM
boundary, and the Pages deploy model are untouched.

## 2. Requirements that must survive (regression guard)

| # | Invariant | Guarded by |
|---|---|---|
| R1 | Byte-identical **core** output (cell grid) | golden frames + FR-6.3/FR-7.4 parity tests |
| R2 | Data-only WASM boundary, **no `web-sys`** (V4a) | code review; module API unchanged |
| R3 | Cell/glyph model, 3 presenters, live editor, presets (FR-5.x) | existing e2e |
| R4 | GitHub Pages deploy, **no custom HTTP headers**, NFR-7 size budget | deploy job; size check |
| R5 | Loads on the baseline the demo already supported | NFR-19 (restored) |

## 3. Key decisions

| # | Decision | Rationale (report 14 §) |
|---|---|---|
| M1 | **Restore universal load**: ship a **scalar-fallback wasm** beside the SIMD one and pick at runtime via a `WebAssembly.validate` SIMD probe. Fixes the W5 `+simd128` regression (blank page on iOS < 16.4). **P0.** | §1 |
| M2 | **Profile-gate the big presenter work**: measure render-vs-present split on a real mid-range iPhone (Web Inspector) before committing to the WebGL2 presenter — mirror the FR-7.1 discipline. | §2 |
| M3 | **Optimize the Canvas2D presenter in place first**: batch the per-cell `source-in` tint into **one whole-frame composite** (bit-identical output), cap DPR, integer coords, canvas-area guard. | §3, §4 |
| M4 | **Adaptive resolution + mobile frame cap** (~30 fps) + pause on `visibilitychange`. Fewer cells is a quality knob, not a feature loss. | §5 |
| M5 | **OffscreenCanvas + Worker** render path with a **main-thread fallback** (iOS 16.4+). | §6 |
| M6 | **WebGL2 presenter is ADDITIVE** — selected at runtime with a Canvas2D fallback, never replacing it. **Gated by M2**; may split to v2.3. WebGPU deferred (iOS 26+). | §7 |
| M7 | **WASM threads deferred** — Pages can't set COOP/COEP; only a service-worker shim works, low payoff on 2-core mobile. | §8, report 10 |
| M8 | **No output-changing shortcuts** — color-quantized glyph caching is rejected because it alters output (R1). | §9 |

## 4. Phase 9 — Compatibility: restore universal load (P0)

**Deliverable:** the demo instantiates and renders on a non-SIMD baseline again, while keeping
the SIMD speedup where supported.

| ID | Requirement |
|---|---|
| FR-9.1 | `web/build.sh` emits **two** artifacts: `tte_wasm_bg.wasm` (`+simd128`) and a scalar `tte_wasm_bg.scalar.wasm` (no SIMD target-feature; `tte-core` `simd` off). Both pass `wasm-opt` and the size check. |
| FR-9.2 | Loader (`app.js`) runs a tiny `WebAssembly.validate(<simd probe bytes>)` and `init()`s the SIMD module when supported, else the scalar one. One shared JS glue where possible; document the wasm-bindgen output layout. |
| FR-9.3 | A **forced-scalar load path** is exercised in CI (load the scalar artifact explicitly) so the fallback can't silently rot. |

### Phase 9 non-functional

| ID | Requirement | Verification |
|---|---|---|
| NFR-19 | The demo boots + renders with the **scalar** module (the iOS < 16.4 path). | CI smoke against the scalar build |
| NFR-20 | **Each** shipped wasm ≤ 250 KB raw (NFR-7 per artifact). | `web/build.sh` size check, both files |

## 5. Phase 10 — Canvas2D presenter performance

**Deliverable:** a measurably smoother mobile demo with **pixel-identical** output to today's
presenter.

| ID | Requirement |
|---|---|
| FR-10.1 | **Done.** Replaced the per-cell `globalCompositeOperation="source-in"` loop with a **single whole-frame composite** (ink layer of white glyphs → one `source-in` with the run-batched fg-colour field → over the bg). The old per-cell path is kept as `GridRenderer.drawCompat` and a Playwright **pixel-equality** test (`scripts/pixel-parity.mjs`, NFR-22) asserts the two are byte-identical. |
| FR-10.2 | Configurable **DPR cap** (default ≤ 2) decoupled from `devicePixelRatio`; assert backing area < the iOS canvas cap (16.7 M px pre-iOS-18). |
| FR-10.3 | **Adaptive internal resolution** driven by measured FPS, a **mobile rAF cap** (~30 fps), and **pause on `visibilitychange`**. All within the cell model. |
| FR-10.4 | **OffscreenCanvas + Worker** render/present path (the worker owns wasm + canvas), feature-detected with a **main-thread fallback**. |

### Phase 10 non-functional

| ID | Requirement | Verification |
|---|---|---|
| NFR-21 | Demonstrated mobile FPS improvement vs the current presenter on the M2 target device (before/after), or a throttled headless proxy. | manual on-device + Playwright (CPU-throttled) |
| NFR-22 | Presenter output **pixel-identical** (FR-10.1) — a canvas pixel-equality check (new vs reference). | Playwright pixel diff |

## 6. Phase 11 — WebGL2 presenter (stretch; M2-gated, candidate v2.3)

**Deliverable:** an additive GPU presenter that offloads cell compositing while keeping the
CPU rasterizer.

| ID | Requirement |
|---|---|
| FR-11.1 | A **WebGL2** presenter: per-cell glyph index + fg/bg uploaded as textures, drawn with one instanced/fullscreen shader sampling a glyph-atlas texture. Auto-selected at runtime with a **Canvas2D fallback** (presenter interface kept identical). |
| FR-11.2 | Visual parity with the Canvas2D presenter within a defined tolerance (GPU sampling need not be bit-identical; acceptance = no perceptible glyph/colour drift on the golden scenes). |

### Phase 11 non-functional

| ID | Requirement | Verification |
|---|---|---|
| NFR-23 | Loads + renders on iOS 15+ (WebGL2); falls back cleanly to Canvas2D when WebGL2 is unavailable. | CI + manual |

## 9. Risks

1. **Batched-composite pixel identity** (FR-10.1) — glyph-edge antialiasing must match. *Mitigation:* the atlas glyphs are already antialiased white masks; `source-in` by a solid per-cell colour reproduces identical edges. The NFR-22 pixel-diff test is the gate; fall back to the per-cell path if any cell diverges.
2. **OffscreenCanvas iOS quirks / older iOS** — *Mitigation:* feature-detect; main-thread fallback (FR-10.4) is mandatory, not optional.
3. **WebGL2 presenter scope creep** (Phase 11) — *Mitigation:* M2-gated and additive; can ship v2.2 without it (split to v2.3).
4. **Forced-scalar CI coverage** — node/Playwright may always support SIMD, hiding fallback rot. *Mitigation:* FR-9.3 loads the scalar artifact *explicitly* rather than relying on a non-SIMD runtime.
5. **DPR cap softening glyphs** — *Mitigation:* cap is configurable; default chosen to balance crispness vs bandwidth; desktop unaffected (already ≤2 typically).
